# Contexto Técnico del Proyecto: Gestión-Alumnos (Haspenrg)
Este documento centraliza el estado actual, las reglas coercitivas de desarrollo y el mapa de intervención técnica del Ecosistema de Gestión Escolar del Colegio HASPEN. Es obligatorio leer y respetar este archivo antes de realizar cualquier cambio en el código.

🚨 Reglas Obligatorias de Trabajo (Mandatorias)
Diagnóstico Técnico Previo: Antes de entregar cualquier propuesta de código, la IA debe desglosar los componentes afectados, las relaciones funcionales, las causas probables del problema y la estrategia de solución.
Prohibición Total de Archivos Largos: Queda estrictamente prohibido enviar scripts completos si superan las 120 líneas. Nos manejaremos únicamente por módulos específicos o bloques atómicos inyectables por parches precisos para evitar fallas de renderizado en el chat.
Mecanismo de Inyección por Parches: Indicar la línea exacta o la función de referencia para que el desarrollador pueda insertar el código mediante búsquedas rápidas (Ctrl + F).
Preservación Estructural: Queda terminantemente prohibido alterar IDs, nombres de clases preexistentes o esquemas CSS responsivos establecidos en las maquetaciones originales de los layouts.
