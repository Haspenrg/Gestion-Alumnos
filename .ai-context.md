# Contexto del Proyecto: Gestión Alumnos (Colegio HASPEN)

## 1. Arquitectura Tecnológica y Reglas de Diseño

- **Frontend**: HTML5, CSS3 nativo ( estilos.css ) y JavaScript asíncrono (Módulos ES6+).
- **Diseño Unificado (Mandatorio)**: Todas las pantallas del ecosistema deben heredar el fondo con el degradado institucional estricto: background: linear-gradient( 135 deg, # a4c2e6 0%, # f7e6d5 50%, # fbc280 100%) !important;
- **Estructuras de Contenido**: Las tarjetas principales, contenedores de formularios y paneles de tablas deben mantener un fondo blanco puro ( #ffffff ) para asegurar el contraste de lectura y una estética UI profesional.
- **Cabeceras Flexibles**: Los headers de los módulos deben implementar Flexbox horizontal ( display: flex; justify-content: space-between; align-items: center; ) para incluir el texto a la izquierda y el escudo chico transparente ( logo1.png con height: 65px; width: auto; ) alineado al extremo superior derecho de la tarjeta.

## 2. Persistencia y Backend (Cloud Firebase)

- **Autenticación**: Firebase Auth. El login se procesa ingresando el DNI, buscando el correo real en la colección "usuarios" de Firestore para ejecutar el inicio de sesión. No se permiten correos sintéticos ( @haspen.edu.ar ).
- **Persistencia Híbrida**: Firestore actúa como la fuente única de verdad para cuentas, perfiles, cursos y configuraciones base. El almacenamiento local ( localStorage ) queda relegado estrictamente a sesiones activas ( 'usuarioActivo' ) e inscripciones temporales.
- **Estándar de Conexión Anti-CORS (Crítico)**: Para evitar bloqueos de seguridad del navegador bajo servidores de desarrollo locales ( Live Server ), todos los scripts que conecten a Firestore deben usar encapsulamiento asíncrono anónimo autoejecutable desarmando la URL de la CDN en caracteres individuales: ( async function() { 'use strict'; const b = 'h' + 't' + 't' + 'p' + 's' + ':' + '/' + '/' + 'w' + 'w' + 'w' + '.' + 'g' + 's' + 't' + 'a' + 't' + 'i' + 'c' + '.' + 'c' + 'o' + 'm' + '/f' + 'i' + 'r' + 'e' + 'b' + 'a' + 's' + 'e' + 'j' + 's' + '/10.12.0/'; const { db } = await import('./firebase-config.js'); const { doc, getDoc, setDoc, collection, getDocs, deleteDoc } = await import( b + 'firebase-firestore.js'); // Lógica del script... })();

## 3. Control de Acceso Basado en Roles (RBAC Adaptativo)

- **Administrador**: Acceso total nativo e inmediato. Habilita dinámicamente los "Paneles de Expansión Institucional" para inyectar semillas de datos (divisiones, modalidades) en caliente hacia Firestore.
- **Tolerancia de Cuerpos**: El script de validación de perfil debe evaluar strings parciales usando .includes("admin") en lugar de igualdades estrictas para prevenir fallos visuales por variaciones tipográficas en la base de datos.
- **Preceptores / Directivos / Profesores**: Se rigen bajo restricciones físicas perimetrales de elementos del DOM y estados de Solo Lectura parametrizados según la matriz guardada.

## 4. Reglas Obligatorias de Trabajo para el Asistente IA

1. **Diagnóstico Técnico Previo**: Desglosar componente afectado, relacionales, diagnóstico y estrategia antes de entregar código.
2. **Prohibición de Código Fragmentado**: Entregar siempre los archivos modificados de manera completa, de punta a punta. Prohibido usar comentarios de elisión ( // ... resto del código ).
3. **Generación Asíncrona por Bloques**: Si un script o layout supera las 500 líneas de código, se segmentará en partes lógicas y se detendrá la respuesta, esperando la confirmación explícita del usuario (*"Dame la parte X"*) para continuar.
4. **Preservación Estructural**: Prohibido alterar IDs, nombres de clases preexistentes o estructuras HTML definidas por el usuario.
