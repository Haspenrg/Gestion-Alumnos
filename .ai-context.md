# Contexto del Proyecto para el Asistente IA

## Información General
* **Nombre:** Gestión de Alumnos - Colegio HASPEN
* **Tecnologías:** HTML5, CSS3 (estilos.css) y JavaScript vanila (ES6). Desarrollo local sin frameworks.
* **Ubicación:** Desarrollo local en VS Code. Persistencia basada en localStorage.

## Estructura Actual del Código
1. **index.html:** Pantalla de Login (Usuario, Contraseña, logo.jpg). Contiene campos numéricos nativos (`pattern="[0-9]+"`) y una ventana modal independiente (`#modalRecuperar`) para restablecimiento autónomo de claves por DNI. Redirige a panel.html.
2. **panel.html:** Panel principal con grilla de 7 módulos controlados por la función `bloquearModulo()`. Si el usuario no tiene permisos, se remueve el `onclick`, se aplica opacidad y se detiene la propagación de eventos:
   - Gestión de Usuarios (id: `mod-usuarios`, vínculo a usuarios.html)
   - Inscripción Alumnos (id: `mod-inscripcion`, alerta pendiente)
   - Crear Cursos (id: `mod-cursos`, alerta pendiente)
   - Materias (id: `mod-materias`, alerta pendiente)
   - Calificaciones y Boletín (id: `mod-calificaciones`, alerta pendiente)
   - Control de Previas (id: `mod-previas`, alerta pendiente)
   - Estadísticas (id: `mod-estadisticas`, alerta pendiente)
3. **usuarios.html:** Interfaz del Administrador para dar de alta y listar usuarios. Incluye contenedores dinámicos ocultos (`#grupoCursosPreceptor` y `#grupoAsignacionProfesor`) para asignaciones específicas según el rol.
4. **usuarios.js:** Lógica para mostrar/ocultar campos dinámicos mediante `configurarCamposRequeridos()`, alternando el atributo `required`. Gestiona la bolsa de horas del Profesor acumulando cadenas de texto en el array `catedrasTemporales`.
5. **estilos.css:** Estilos unificados para el login, la grilla adaptable del panel (1 col en móvil, 2 en tablet, 4 en PC mediante `.dashboard-grid`) y el formulario de usuarios.
6. **app.js:** Inicializa la base local bajo la clave `usuariosColegio` mediante un array semilla (`usuariosSemilla`). Gestiona la validación local de credenciales, almacena la sesión en `usuarioActivo` y procesa la actualización de claves en caliente.

## Definición de Roles y Permisos (Establecido)
* **Administrador:** Control total. Único que accede a usuarios.html para gestionar cuentas, asignar responsabilidades y resetear claves al DNI.
* **Directivo:** Acceso a todo el panel principal para supervisión y estadísticas, excepto al módulo de Usuarios (`mod-usuarios`).
* **Preceptor:** Gestión operativa diaria. Se le bloquea el módulo de Usuarios. Se le asignan de forma obligatoria exactamente 2 cursos de entre los 36 posibles (1º a 6º año, Div. A a F) mediante selectores específicos.
* **Profesor:** Perfil pedagógico limitado. Se le bloquean 5 módulos administrativos. Posee una "bolsa de horas" dinámica que le permite tener asignadas múltiples cátedras (Curso + Materia) gestionadas por etiquetas `.catedra-tag`. Va directo a su pantalla de notas al loguearse.
* **Coordinación:** Perfil de supervisión intermedio. Se le bloquean 5 módulos administrativos (`mod-usuarios`, `mod-inscripcion`, `mod-cursos`, `mod-materias`, `mod-previas`). Posee acceso exclusivo dentro del panel principal únicamente a los módulos de Calificaciones y Boletín (`mod-calificaciones`) y Estadísticas (`mod-estadisticas`).

## Reglas de Trabajo
* No generar código por generar; trabajar estrictamente sobre la tarea solicitada.
* Mantener el diseño limpio, institucional y coherente con los estilos CSS existentes.
* El sistema debe permitir licencias docentes (dos usuarios pueden compartir una misma cátedra asignada).
