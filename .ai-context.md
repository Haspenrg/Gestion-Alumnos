# Contexto Consolidado del Sistema - Gestión de Alumnos (Colegio HASPEN)

## 📌 Protocolo de Ingeniería y Reglas Operativas Obligatorias

### Regla N° 1: Diagnóstico Técnico Previo
Antes de escribir cualquier línea de código, la IA debe emitir un diagnóstico técnico detallado que incluya:
1. Componente afectado.
2. Relacionales cruzados (a qué otros archivos o estructuras afecta).
3. Diagnóstico del problema o justificación del cambio.
4. Estrategia de intervención.

### Regla N° 2: Prohibición Absoluta de Código Fragmentado
Queda estrictamente prohibido entregar respuestas con marcadores de posición, comentarios de elisión (ej: `// ... resto del código ...`) o funciones aisladas si el archivo requiere una actualización estructural. El código modificado debe presentarse completo de punta a punta para garantizar la integridad de la sintaxis, sujeto a los límites de la Regla N° 3.

### Regla N° 3: Generación Asíncrona Orientada a Bloques (Scripts > 500 líneas)
Si el código final a generar posee o excederá las 500 líneas de desarrollo, la IA tiene la obligación estricta de segmentar la entrega en partes consecutivas numeradas (ej: Parte 1 de 2, Parte 2 de 2). 
* La IA detendrá la escritura al final de la primera parte en un punto de quiebre limpio (fuera de funciones o bloques lógicos abiertos).
* Esperará la confirmación explícita del usuario ("Dame la parte 2") antes de proceder a la impresión del siguiente bloque.
* Esto previene de forma absoluta los cortes abruptos de sintaxis por límites de tokens de salida.

### Regla N° 4: Preservación Estructural Absoluta
No se alterará, renombrará ni modificará ninguna etiqueta HTML (`<label>`, `<input>`, `id`, `class`, etc.), nombre de campo o estructura CSS preexistente en los layouts originales provistos por el operador, salvaguardando la total compatibilidad con el código actual de producción. Cualquier adición de funcionalidad se inyectará como un bloque complementario manteniendo de forma matemática los estilos, tipografías y densidad visual original.

## 🛠️ Stack Tecnológico Establecido
* **Frontend Responsivo / Mobile-First:** HTML5 Plano y CSS3 Plano. Los layouts deben diseñarse de forma flexible (Flexbox/Grid) y utilizar Media Queries para garantizar una visualización, carga de datos y navegación óptima tanto en computadoras de escritorio de preceptoría como en tablets y teléfonos celulares.
* **Lógica:** JavaScript Vanilla (ES6+) estructurado de manera limpia, libre de dependencias externas en su núcleo.
* **Persistencia y Arquitectura de Datos:** Almacenamiento local temporal basado en `localStorage`. Toda la lógica de persistencia asíncrona mediante promesas y JSON debe diseñarse de forma modular y desacoplada, facilitando una migración directa y transparente en el corto plazo hacia una base de datos en la nube (como Firebase Firestore / Realtime Database).
