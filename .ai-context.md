# Contexto Técnico del Proyecto: Gestión-Alumnos (Haspenrg)

Este documento centraliza el estado actual, las reglas coercitivas de desarrollo y el mapa de intervención técnica del Ecosistema de Gestión Escolar del Colegio HASPEN. **Es obligatorio leer y respetar este archivo antes de realizar cualquier cambio en el código.**

---

## 🚨 Reglas Obligatorias de Trabajo (Mandatorias)

1. **Diagnóstico Técnico Previo:** Antes de entregar cualquier propuesta de código, la IA debe desglosar los componentes afectados, las relaciones funcionales, las causas probables del problema y la estrategia de solución.
2. **Prohibición de Código Fragmentado:** Queda estrictamente prohibido entregar respuestas con elisiones (`// ... resto del código`). Todo archivo intervenido debe ser provisto de manera completa, de punta a punta, para asegurar que no se introduzcan regresiones de sintaxis o fallas en el parser del chat.
3. **Generación Asíncrona por Bloques:** Si un script o layout supera las 500 líneas de código, se segmentará en partes lógicas o secciones de texto plano muy cortas (menores a 80 líneas) utilizando técnicas de concatenación u ofuscación de strings para garantizar la inmunidad ante fallas de renderizado del DOM del chat.
4. **Preservación Estructural:** Queda terminantemente prohibido alterar IDs, nombres de clases preexistentes o esquemas CSS responsivos establecidos en las maquetaciones originales de los layouts.

## 🛠️ Estado Actual del Sistema y Cambios Implementados Hoy

### 1. Módulo: Gestión de Usuarios (`usuarios.html` y `usuarios.js`)
* **Mejora en Carga Inicial:** Se reestructuró el flujo asíncrono de inicialización con bloques `try/catch` aislados. Se le dio prioridad máxima a `inicializarSemillaUsuarios()` sobre la carga de catálogos dinámicos para evitar bloqueos en cascada cuando la base de datos de Firestore se encuentra completamente vacía.
* **Módulo de Contraseñas Personalizadas:** Se integraron con éxito los campos `claveUsuario` y `confirmarClaveUsuario` en el DOM y en la interfaz. El Administrador ahora puede asignar claves primarias personalizadas.
* **Validación de Seguridad:** Se configuró una validación en caliente que exige coincidencia exacta y un mínimo de 6 caracteres reglamentarios para altas en Firebase Auth, omitiéndose de forma segura durante las ediciones para preservar claves vigentes.
* **Alineación de Autenticación:** Se corrigió la sintaxis modular de Firebase v10 (`createUserWithEmailAndPassword`), inyectando de manera correcta el parámetro de la instancia secundaria aislada (`secondaryAuth`) para registrar operarios en la nube sin interrumpir la sesión del Administrador logueado.

### 2. Módulo: Legajo Digital del Alumno (`inscripcion.html` y `inscripcion.js`)
* **Datos Adicionales del Adulto Responsable:** Se expandió el formulario y el modelo relacional inyectando las variables de captura y persistencia para el **DNI del Tutor** (`dniTutorAlumno`) y el **Correo Electrónico del Tutor** (`emailTutor`).
* **Flexibilización de Entrada:** Siguiendo directivas de conectividad familiar, el campo del correo electrónico del tutor se configuró como un parámetro **opcional** eliminando la restricción coercitiva `required` de la vista HTML.
* **Ciclo de Edición Completo:** Se actualizaron de forma quirúrgica las funciones `guardarLegajoDigital`, `cargarLegajoEnFormulario` y la rutina de limpieza `salirModoEdicion` para asegurar que el DNI y el Email del Tutor se resguarden en Firestore, se vuelquen en la UI al editar y se vacíen al cancelar.

## 🔍 Pendiente Actual: Automatización de CUIL (Falta Resolver)

Quedó diseñado y estructurado el camino técnico para implementar el **autocompletado algorítmico del CUIL basado en el algoritmo matemático oficial de ANSES**, pero la integración final quedó en pausa.

### Estado en el que quedó el código:
* **Fallas Detectadas:** El autocompletado no se dispara de forma automática en los casilleros de la interfaz web al ingresar los documentos, indicando posibles errores de orden de compilación de las dependencias o desajustes menores en los identificadores de referencia de la vista.

### Tareas Críticas Pendientes para la Próxima Sesión:
1. **Unificación Jerárquica:** Mover y empaquetar el motor algorítmico `calcularCuilAutomatico(dniStr, generoStr)` en la parte superior de los escuchadores del script para garantizar que la función esté compilada antes de ser invocada por los detectores de tipeo en espejo.
2. **Control de IDs de Género:** Auditar el archivo `inscripcion.html` para asegurar que los elementos selectores posean exactamente los IDs `generoAlumno` y `generoTutor` exigidos por las constantes del DOM.
3. **Vinculación de Campos de Salida:** Verificar la existencia de las cajas receptoras (`cuilAlumno` y `cuilTutor`) con su propiedad `readonly` para permitir el llenado algorítmico automático e instantáneo (Varón: prefijo 20 / Mujer: prefijo 27) tanto para el estudiante como para el Adulto Responsable.
4. **Sincronización en la Nube:** Completar las adiciones de guardado para la propiedad `cuilTutor` dentro del payload JSON enviado a Cloud Firestore.
